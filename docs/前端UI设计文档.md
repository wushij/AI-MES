## 1 设计规范

> 本节基于 `docs/模仿代码/frontend/styles/theme.css` 与 `utils/theme.ts` 实现，采用 **CSS 变量 + 运行时主题切换**，全量覆写 Element Plus 皮肤。

### 1.1 主题色切换系统

#### 设计原理

- **CSS 变量令牌**：在 `theme.css` 中定义 `--cd-primary`、`--theme-primary-muted` 等多级色彩变量，并映射到 Element Plus 组件（按钮、输入框、表格等）。
- **运行时计算配色**：`theme.ts` 中通过 `darken()`、`tintBg()`、`rgba()` 根据任意主色实时计算 hover、激活、淡化背景色。
- **DOM 应用**：调用 `applyThemePreset(id)` 或 `applyCustomTheme(hex)`，通过 `document.documentElement.style.setProperty` 写入根节点。

#### 预设主题（AI-MES 推荐 `indigo`）

| ID | 名称 | 主色 | 页面背景 | 侧边栏背景 |
|----|------|------|----------|------------|
| `indigo` | 靛蓝 **（默认）** | `#4f46e5` | `#f5f3ff` | `#ffffff` |
| `slate` | 石墨 | `#010710` | `#f3f5f8` | `#ffffff` |
| `teal` | 青绿 | `#0d9488` | `#f0fdfa` | `#ffffff` |
| `emerald` | 翠绿 | `#059669` | `#ecfdf5` | `#ffffff` |

```vue
<el-progress
```

### 5.2 验证码按需触发

> 参考：`docs/模仿代码/backend/security/LoginProtectionService.java` + `Login.vue` 中 `syncCaptchaState()`

#### 触发规则

| 场景 | 是否显示验证码 |
|------|----------------|
| 首次登录（IP 失败 < 3 次） | 否 |
| IP 累计失败 ≥ 3 次 | 是 |
| 登录失败后 | 重新检测，可能需要验证码 |

#### 前端流程

```
页面挂载 / 登录失败
    ↓
GET /api/auth/captcha/required
    ↓
required === true → showCaptcha = true → GET /api/auth/captcha 刷新图片
required === false → 隐藏验证码区域
    ↓
POST /api/auth/login { username, password, captchaId?, captchaAnswer? }
```

#### 后端防护策略（Redis 缓存）

| 维度 | 阈值 | 动作 |
|------|------|------|
| IP 失败 | ≥ 3 次 | 强制验证码 |
| IP 失败 | ≥ 30 次 | 封禁 IP 60 分钟 |
| 账号失败 | ≥ 5 次 | 锁定账号 15 分钟 |

### 5.3 交互逻辑

- 密码框 suffix 眼睛按钮切换明文/密文
- Enter 键提交；验证码区 Enter 同样触发
- 登录中按钮 Loading + 禁用防重复提交
- 失败：`ElMessage.error` + 重新 `syncCaptchaState()`
- 成功：管理员 → `/admin/users`，其余 → `/dashboard`

## 16 交互状态与反馈

### 16.1 Loading 状态

| 场景 | 实现方式 |
|------|----------|
| 页面初始加载 | 表格行骨架屏（3 行 Skeleton） |
| 按钮操作中 | `<el-button :loading="true">` |
| AI 客服等待回复 | 气泡内三点跳动动效 |
| AI 排产分析中 | 全区域骨架屏 + 进度提示文字 |
| 全局路由切换 | 顶部进度条（NProgress 效果） |

### 16.2 操作反馈

| 操作 | 反馈方式 |
|------|----------|
| 表单提交成功 | `ElMessage.success('操作成功')` 右上角 Toast |
| 表单校验失败 | 字段下方红色提示文字 |
| 危险操作 | `ElMessageBox.confirm` 确认弹窗 |
| 删除成功 | 表格行消失动效 + success Toast |
| 网络请求失败 | `ElMessage.error('网络异常，请重试')` |
| Coze 超时 | 聊天框内显示「AI 响应超时，请重试」 |
| 表单提交中 | 提交按钮变为 Loading + 禁用，防止重复提交 |

### 17.2 Axios 拦截器配置

```typescript
// api/request.ts 要点
const instance = axios.create({
  baseURL: '/api',
  timeout: 30000,
})

// 请求拦截：自动携带 token
instance.interceptors.request.use(config => {
  const token = useUserStore().token
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截：统一处理 401（跳登录）、403（无权限）
instance.interceptors.response.use(
  res => res.data,
  err => {
    if (err.response?.status === 401) router.push('/login')
    if (err.response?.status === 403) router.push('/403')
    ElMessage.error(err.response?.data?.message || '请求失败')
    return Promise.reject(err)
  }
)
```

### 17.2 主要 Pinia Store

```typescript
// stores/user.ts
export const useUserStore = defineStore('user', {
  state: () => ({
    token: localStorage.getItem('token') || '',
    userInfo: null as UserInfo | null,
    role: '' as 'admin' | 'supervisor' | 'worker' | '',
  }),
  getters: {
    isAdmin: s => s.role === 'admin',
    isSupervisor: s => s.role === 'supervisor',
    isWorker: s => s.role === 'worker',
  },
})
```
