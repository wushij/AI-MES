## 二、题目要求（原始题干）

### 2.1 应用范围

设计一套以**生产计划、工序状态、班组任务、异常停机、缺料预警**为核心的车间管理应用。

### 2.2 必选 AI 能力

至少接入 **1 个 Coze 智能客服智能体**，用于处理：

- 生产进度查询
- 异常处理问答
- 操作规程（SOP）咨询
- 班组任务说明

### 2.3 推荐扩展

建议再增加 **1 个智能排产 / 产能分析智能体**，用于提供：

- 订单优先级建议
- 瓶颈分析
- 任务派发建议

### 2.4 前端要求

至少包含以下页面/模块：

| 模块 | 说明 |
|------|------|
| 生产看板 | 展示计划进度、工单状态、关键指标 |
| 任务列表 | 班组/员工可查看、领取、反馈任务 |
| 异常反馈表单 | 上报停机、缺料、质量等异常 |
| 客服入口 | 智能客服对话界面 |

### 2.5 后端要求

至少实现：

- 业务数据存储（本地数据库或持久化文件）
- 接口转发（封装 Coze API，不暴露密钥）
- 智能体返回结果展示（结构化或文本回显）

---

## 三、统一技术要求

以下要求适用于全部选题，题目二必须全部满足。

1. **场景真实**：工业场景合理，业务流程清晰，有明确业务对象与用户角色。
2. **智能体接入**：至少 1 个 Coze 智能体用于客服；鼓励 2 个及以上智能体（分析、工单、排产等）。
3. **完整实现**：必须完成「前端页面 + 后端接口 + Coze API 对接」全链路，**不允许**仅静态页面或仅 Coze 编排无后端。
4. **功能边界**：
   - **前端**：数据展示、表单录入、结果回显、客服入口
   - **后端**：API 封装、异常处理、日志、本地数据存储
5. **可复现**：系统可运行，或可通过录屏演示完整闭环：

```
前端发起请求 → 后端转发 Coze → Coze 返回结果 → 前端展示
```

## 六、项目需求分析（AI-MES）

结合题目二要求，本项目定位为 **AI-MES 智能生产车间管理平台（AI Manufacturing Execution System）**。

### 6.1 项目背景

传统生产车间在生产计划、工单、设备与数据分析等方面存在信息孤岛、人工统计效率低、异常处理不及时等问题。将 AI 大模型与 MES 结合，可实现智能问答、智能排产、数据分析与异常诊断，提升生产效率与管理水平。

### 6.2 项目目标

- 实现生产计划数字化管理
- 实现生产工单全生命周期管理
- 实现车间班组协同管理
- 实现设备运行状态管理
- 实现生产进度实时统计
- 实现异常事件统一管理
- 实现生产数据可视化分析
- 接入 AI 智能助手：智能问答、智能排产、生产分析
- 降低人工管理成本，提高决策效率

### 6.3 用户角色

| 角色 | 职责 |
|------|------|
| 管理员 | 系统配置、用户管理、权限分配、平台维护 |
| 车间主管 | 生产计划制定、工单分配、设备管理、生产监控、数据统计 |
| 普通员工 | 查看生产任务、反馈进度、上报异常 |

### 6.4 核心业务流程

```
登录系统
    ↓
首页驾驶舱（生产看板）
    ↓
制定生产计划
    ↓
生成生产工单
    ↓
班组领取任务
    ↓
开始生产 → 实时反馈进度
    ↓
异常上报 → 主管处理
    ↓
生产完成
    ↓
AI 分析生产数据 → 生成统计报表
```

## 八、技术架构

### 8.1 总体架构

```
┌─────────────────────────────────────────────────────────┐
│                      前端（Vue3）                        │
│  驾驶舱 │ 工单 │ 计划 │ 异常表单 │ AI 客服 │ 排产建议    │
└──────────────────────────┬──────────────────────────────┘
                           │ HTTP / WebSocket
┌──────────────────────────▼──────────────────────────────┐
│                   后端（Spring Boot 3）                  │
│  认证 │ 业务 API │ 数据持久化 │ Coze 代理 │ 日志/异常    │
└──────────────┬──────────────────────────┬───────────────┘
               │                          │
        ┌──────▼──────┐            ┌──────▼──────┐
        │ MySQL/Redis │            │  Coze 平台   │
        │  业务数据    │            │ 智能体 API   │
        └─────────────┘            └─────────────┘
```

### 8.2 技术选型

| 层次 | 技术 |
|------|------|
| 前端 | Vue3、TypeScript、Vite、Element Plus、Pinia、Vue Router、Axios、ECharts |
| 后端 | Spring Boot 3、MyBatis-Plus、MySQL、Redis、Sa-Token、Knife4j |
| AI | OpenAI Compatible API / DeepSeek / **Coze 智能体** |
| 部署 | Docker、Docker Compose、Nginx |

### 8.3 API 对接闭环（考核重点）

```
1. 用户在前端「AI 客服」输入：「查询工单 WO-001 进度」
2. 前端 POST /api/ai/chat { message, sessionId, context? }
3. 后端读取本地工单数据，组装 Prompt/Workflow 参数
4. 后端调用 Coze API（密钥仅存服务端）
5. Coze 返回自然语言或结构化 JSON
6. 后端解析响应，可选合并本地业务数据
7. 前端展示 AI 回复，必要时附带工单卡片/进度条
```

**排产智能体闭环（扩展）：**

```
前端提交排产请求 → 后端聚合工单/设备/物料数据
→ 调用 Coze 排产 Workflow → 返回优先级与派发建议 → 前端表格/图表展示
```

## 九、前后端页面与接口规划

### 9.1 前端页面

| 页面 | 路由示例 | 核心元素 |
|------|----------|----------|
| 登录页 | `/login` | 账号密码、角色跳转 |
| 首页驾驶舱 | `/dashboard` | KPI 卡片、进度图表、异常列表、快捷入口 |

| 工单管理 | `/work-orders` | 任务列表、状态筛选、领取/完成操作 |
| 班组管理 | `/teams` | 班组与成员、任务分配 |
| 异常管理 | `/exceptions` | 异常反馈表单、处理记录 |
| AI 智能客服 | `/ai-chat` 或全局浮窗 | 对话历史、快捷问题、结果展示 |
| 智能排产（扩展） | `/ai-scheduling` | 输入条件、排产建议展示 |

### 9.2 后端接口（示例）

| 模块 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 认证 | POST | `/api/auth/login` | 登录 |

| 工单 | GET/POST/PUT | `/api/work-orders` | 工单 CRUD 与状态流转 |
| 班组 | GET | `/api/teams` | 班组与任务 |
| 异常 | POST | `/api/exceptions` | 异常上报与处理 |
| 驾驶舱 | GET | `/api/dashboard/stats` | 统计数据 |
| **AI 客服** | POST | `/api/coze/chat` | 转发 Coze 对话（**必做**） |
| **AI 排产** | POST | `/api/coze/scheduling` | 转发排产 Workflow（推荐） |

## 十、数据设计要点

### 10.1 核心业务实体

| 实体 | 主要字段 |
|------|----------|
| 生产计划 | 计划编号、产品、数量、计划日期、状态 |
| 生产工单 | 工单编号、关联计划、工序、班组、进度、状态 |
| 班组 | 班组编号、名称、成员、负责人 |
| 异常事件 | 异常类型（停机/缺料/质量）、设备/工单、描述、处理状态 |
| 物料预警 | 物料编码、库存、安全库存、预警状态 |
| AI 对话记录 | 会话 ID、用户、消息、回复、时间（便于报告截图） |

### 10.2 工序状态枚举

`待开始` → `进行中` → `已暂停（异常）` → `已完成`

### 10.3 工单状态枚举

`待分配` → `待领取` → `生产中` → `异常处理中` → `已完成`
